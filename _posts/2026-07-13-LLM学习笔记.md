## 一、理解大语言模型

1. LLM源于NLP，是machine learning和AI的子集，关键技术是Transformer，本质是一个有大量参数大量数据的神经网络，通常被称为生成式AI（GenAI）

2. 创建LLM的一般过程包括预训练和微调
   - **预训练**：在一个大型且多样化的数据集上进行训练，以便获得对语言的广泛理解
     - 数据只是普通文本，没有任何标注，通常称为基础模型
   - **微调**：在一个更针对特定任务或领域的数据集上进行专门训练
     - 在优质的标注数据上对LLM进行进一步训练（分类任务微调或指令微调）
     - **全权重微调**：在训练过程中对模型的所有预训练权重进行调整，只会对预训练权重进行微小调整，而不是大幅度改变。这种方式能够让模型保持原有的语言生成能力，同时使其在特定任务上表现得更好。
     - **冻结部分权重的微调**：一般冻结低层（往往是学习到的基础语言特征），对高层的权重进行调整，常在需要加速训练，或者数据量较小，全权重微调可能导致过拟合的情况下使用

3. 原始的Transformer由两部分组成：encoder和decoder
   - 编码器模块处理文本输入，将其编码为一系列数值表示或向量，以捕捉输入的上下文信息
   - 解码器模块利用这些编码向量生成输出文本
   - **自注意力机制**：使模型能够相对地权衡序列中不同单词或标记的重要性，帮助模型捕捉输入数据中的远程依赖关系和上下文关系

4. BERT & GPT
   - **BERT**（Bidirectional Encoder Representations from Transformers）
     - 一种擅长"理解文本"的Transformer Encoder模型
     - MLM（Masked Language Modeling，掩码语言模型），通过"完形填空"学会语言理解
     - 双向看上下文
     - 擅长理解、分类、语义匹配
     - Encoder-only + 双向理解 + MLM预训练
   - **GPT**（Generative Pre-trained Transformer）
     - 一种擅长"根据前文预测下一个Token，并持续生成内容"的Transformer Decoder模型
     - 预测下一个Token
     - Decoder-only Transformer + Causal Attention + Next Token Prediction

5. zero-shot学习指的是在没有先前具体示例的情况下，能够处理完全未见过的任务。而few-shot学习则是指模型可以从用户提供的极少量示例中进行学习

6. 大部分LLM都是基于Transformer来实现的，因此Transformers和LLM常常被视为同义词。但是并非所有的Transformers都是LLM，因为它们也可以用于计算机视觉。同时，并非所有的LLM都是基于Transformers的，市场上也有一些基于递归和卷积架构的大语言模型

7. 模型能够执行未明确训练的任务被称为"涌现行为"

## 二、处理文本数据

1. 将数据转换为向量格式的过程通常被称为嵌入（Embedding），我们可以通过特定的神经网络层或其他预训练的神经网络模型来对不同类型的数据进行嵌入

2. 嵌入的本质是将离散对象映射到连续向量空间中的点

3. 单词嵌入是最常用的文本嵌入形式，但也存在句子、段落或整篇文档的嵌入，常被用于检索增强生成技术（RAG，Retrieval-Augmented Generation）
   - 知识的时效性、模型大小的限制、幻觉
   - 将外部知识库向量化后存入到向量数据库中，当用户提交一个查询时，检索到的信息被作为额外的上下文信息输入到LLM中，LLM会将这些外部信息与原始输入结合起来，以更准确和丰富的内容生成回答
   - RAG并不会让LLM真正"记住"知识，而是在推理时临时把外部知识塞进Context Window

4. Word2Vec，将单词投影到二维空间进行可视化时，可以看到相似的词汇聚在一起，更高的维度可能捕捉到更细微的关系，但代价是计算效率的降低
   - 最小的GPT-2模型（117M和125M参数）使用768维的嵌入大小，而最大的GPT-3模型（175B参数）则使用12,288维的嵌入大小。

5. LLM一般会有一个自己的embedding层，而不是直接使用Word2Vec这种embedding预训练模型。优势在于：能够针对特定的任务和数据进行优化

6. embedding的基本流程：文本分词 → 单词转token → token转向量

7. 文本分词（tokenize）：将输入文本拆分为单个token

8. 可以引入特殊token，给LLM提供更多信息

   | Token | 说明 |
   |-------|------|
   | `[BOS]`（序列开始） | 表示文本的起始位置，指示LLM内容的开始 |
   | `[EOS]`（序列结束） | 位于文本的末尾，在连接多个无关文本时特别有用，类似于 `<\|endoftext\|>` |
   | `[PAD]`（填充） | 在使用大于1的批量大小数据集训练LLM时，为了确保所有文本长度一致，较短的文本会用`[PAD]` token进行扩展或填充（也可以通过掩码矩阵来实现） |
   | `<\|unk\|>`（未知） | 原始文本中不存在的token |

   > GPT采用字节对编码分词器，将单词分解为子词单元，不需要这些东西

9. **Byte Pair Encoding（BPE）**
   - BPE的本质，就是不断把高频相邻符号合并成更大的Token
   - 字符太碎，整词太大，BPE用"高频子词"折中

10. 数据集示例

    ```
    txt: "in the heart of the city, ..."
    window: 4
    input:  ["in", "the", "heart", "of"]
    output: ["the", "heart", "of", "the"]
    ```

    - 读取input[0:0]时，预测到output[0]
    - 读取input[0:1]时，预测到output[1]
    - 读取input[0:2]时，预测到output[2]
    - 读取input[0:3]时，预测到output[3]

11. torch数据集的部分参数

    - **dataset**
      - `max_length`：一个输入矩阵的长度
      - `stride`：步长，从原始txt中取输入数据的步长。批次之间更多的重叠可能会导致过拟合，更大的步长可能会导致信息丢失
    - **dataloader**
      - `batch_size`：每次取的输入矩阵个数
      - `drop_last`：最后一批次小于指定的batch_size时丢弃该批次，以防止训练期间的损失峰值
      - `num_workers`：进程个数

12. 反向传播算法/自动微分？

13. **embedding**：将token_id转换成向量表示
    - `torch.nn.Embedding(num_embeddings, embedding_dim)`：依据传入的个数和维度，生成num个dim维向量，以供token_id查表

14. **位置编码**：从embedding表里面检索到的向量，不包含token的位置信息，会损失一部分信息
    - **绝对位置嵌入**：编码token在文本中的绝对位置
    - **相对位置嵌入**：编码token之间的距离

15. 张量维度示例

    如果batch_size=8，每个批次里面有4个token_id，每个token_id embedding成一个256维的向量，则每次取数据集，会取出一个`8 x 4 x 256`的张量

    ```
    token_id:              (8, 4)
    token_embedding_layer: 50257 * 1 * 256   # 50257是整个词表大小
    token_embedding:       (8, 4, 256)        # 每个token_id查表，转换成1*256的向量

    pos_embedding_layer:   context_len * 256  # context_len是上下文长度
    pos_embedding:         context_len * 256  # 从0到context_len-1，每个数字查表获取位置embedding

    input_embedding = token_embedding + pos_embedding  # (8, 4, 256)
    ```

16. **torch tensor广播加法**：把较短shape在左侧补1，右对齐之后每一维度都相同或者其中一个是1

    ```python
    # 1. ok
    [8, 4, 256] + [4, 256] = [8, 4, 256] + [1, 4, 256]
    # B的同一份[4, 256]数据，会分别加到A的8个[4, 256]切片上

    # 2. ok
    [8, 4, 256] + [4, 1] = [8, 4, 256] + [1, 4, 1]

    # 3. 不支持
    [8, 4, 256] + [4] = [8, 4, 256] + [1, 1, 4]
    ```

    > embedding表是怎么构筑的？随机嘛？

17. **小结**

    ```
    txt
    → 分词(BPE, 空格分词, 特殊token)
    → tokenized_txt
    → token_id(去重排序编号, BPE)
    → 数据采样(滑动窗口, dataloader, dataset)
    → 数据集(input & output)
    → embedding(查表, 位置编码)
    → input_embedding
    ```

## 三、实现注意力机制

1. 简化自注意力机制 → 自注意力机制 → 因果注意力机制 → 多头注意力机制

2. RNN中有长距离依赖问题和梯度消失问题
   - 前一步的输出会作为当前步骤的输入，编码器按顺序处理文本内容，在每个步骤中，编码器会更新其隐状态（即隐藏层的内部值），试图在最终的隐状态中捕捉整个输入句子的含义
   - 解码阶段RNN无法直接访问编码器的早期隐藏状态，因此它只能依赖当前隐藏状态来封装所有相关信息

3. **注意力机制**：在处理每个词时，将该词与句子中的每个其他词进行相似度计算，判断应该关注哪些词，不需要依赖所有的中间隐藏状态

4. 2014年，研究人员尝试在RNN编码解码架构中引入注意力机制；2017年，自注意力机制被提出

5. **self的含义**：传统机制关注的是两个不同序列间的关系，自注意力机制通过关联同一输入序列中的不同位置来计算注意力权重

6. 简化自注意力的计算
   - 每个token向量与其他token向量点乘，得到一个一维向量，称为注意力权重（相关性）
     - 可以用矩阵`@`转置矩阵来快速计算（`@`表示矩阵乘法），得到一个N阶方阵
   - 归一化（`torch.softmax`）
     - 二维张量（例如形状为`[3, 4]`），`dim=-1`将沿列方向进行归一化，使得每一行的值（沿列方向求和）之和等于1
   - 以omega的每一项作为权重，把所有token向量加权相加，得到上下文向量（加权求和）
     - 也可以用`@`计算

7. 带有可训练权重的自注意力机制
   - 每个token，通过WQ、WK、WV三个权重矩阵，计算出对应的QKV向量
     - **Q向量（查询向量）**：查询向量代表了这个词在寻找相关信息时提出的问题
     - **K向量（键向量）**：键向量代表了一个单词的特征，或者说是这个单词如何"展示"自己，以便其它单词可以与它进行匹配
     - **V向量（值向量）**：值向量携带的是这个单词的具体信息，也就是当一个单词被"注意到"时，它提供给关注者的内容
   - 每个token的Q去和其他所有K点乘，得到注意力权重
   - 归一化
   - 每个token的V去用注意力权重加权求和，得到上下文向量
   - 一般GPT内的权重矩阵不会导致token向量维度发生变化
   - **缩放点注意力机制**：当embedding维度较大时，QK点乘的结果可能会很大，这时需要对结果缩放，防止softmax函数劣化，一般的缩放办法是除以维度的平方根

   > WQ、WK、WV三个权重矩阵是怎么初始化的？Linear？

8. **因果自注意力机制**
   - 每个token只能看到自己以及自己之前的token向量
   - 简单的做法是对注意力权重的对角线以上置0，然后重新归一化（与`torch.tril`相乘，注意不是矩阵乘法，是每个对应元素相乘）
   - 更好的做法是对角线以上置为`-inf`，然后直接用softmax
   - **dropout**：计算完注意力权重之后，随机将部分权重丢弃，防止过拟合。其他剩余的权重会等比例放大（`1/(1-p)`，比如50%的dropout率，其他权重会增大一倍）

9. **多头注意力机制**
   - **堆叠自注意力**：使用多个自注意力类，然后将结果拼接（cat）
   - **权重分割**：把每个token向量拆分成多个head向量，head向量先做点乘，计算出QKV，各个head计算注意力权重，然后计算上下文向量，最后将向量拼接
     - 计算过程涉及transpose、view等torch计算

## 四、从零开始实现一个用于文本生成的GPT模型

1. `qkv_bias`参数决定是否在多头注意力的查询、键和值的线性层中加入偏置向量

2. ReLU的作用是将负值设为0，确保输出层中没有负值

3. 数据流

   ```
   txt(字符串)
   → tokenizer(tiktoken(gpt2))
   → token_id矩阵  int(2 * 4)
   → token_embedding + pos_embedding  2 * 4 * 768
   → drop_embedding  2 * 4 * 768
   → TransformerLayers  2 * 4 * 768
       → 归一化
       → 多头注意力
       → dropout
       → 快捷连接
       → 归一化
       → 前馈网络
           → 扩维线性层 + 激活函数 + 锁维线性层
       → dropout
       → 快捷连接
   → LayerNorm  2 * 4 * 768
   → out_head   2 * 4 * 50257
   → de-tokenizer  int(2 * 4)
   → 字符串  2个字符串，每个字符串4个词，分别是input的每个词的下一个预测
   ```

4. **层归一化**：将神经网络层的激活（输出）调整为均值为0，方差为1，可以加速权重的收敛速度，确保训练过程的一致性和稳定性，通常应用于多头注意力模块的前后以及最终输出层之前

   ```
   计算出均值mean, 方差var
   norm_x = (x - mean) / sqrt(var)
   引入scale, shift, eps...
   ```

5. GELU和SwiGLU是更复杂、平滑的激活函数，分别结合了高斯分布和sigmoid门控线性单元
   - FeedForward = 扩维线性层 + 激活函数 + 锁维线性层
   - 扩展后的高维空间可以让模型"看到"输入数据中更多的隐藏特征，提取出更丰富的信息

6. **快捷连接**：在神经网络的不同层之间插入的连接结构，提升深度神经网络架构的训练性能
   - 也称作跳跃连接或残差连接，最初是在计算机视觉中的深度网络（尤其是残差网络）提出的，用于缓解梯度消失问题
   - 将某层的输出加到后续层的输出上来实现，保持梯度流动

7. **权重共享**：GPT-2架构中使用了一种称为"权重共享"的概念，这意味着GPT-2架构将token嵌入层的权重复用于输出层

8. 文本生成流程

   ```
   input
   → GPT
   → 下一个token
   → 将token加入input
   → 继续生成直到达到用户指定的token数量
   ```

   每一步，模型会输出一个矩阵，其中的向量表示潜在的下一个token

9. 将模型置于`.eval()`模式，禁用训练时使用的随机组件（如dropout）

## 五、在无标记数据集上进行预训练

1. 开发LLM的流程主要包括：模型编码、预训练、微调

2. 文本评估过程的一部分是衡量生成的token与正确预测目标之间的差距，模型训练的目标是提高正确目标token ID所在位置的softmax概率

3. 权重更新通过一种称为反向传播的过程来实现，需要一个损失函数，该函数用于计算模型预测输出与实际目标输出之间的差异

4. LLM中对概率的计算通常使用对数，对数的累加性质允许我们将原本的累乘操作转换为累加，使得计算更加稳定和高效

5. **交叉熵损失**：计算损失函数

6. 损失函数计算过程
   - 对每次input的期望target所在token的概率求对数
   - 所有对数值求平均（一般是一个负的值）
   - 平均值作为评估值，目标是平均值降低为0

7. 训练流程：从迭代各个epoch开始，处理批次数据、重置和计算梯度、更新权重，最后进行监控步骤如打印损失和生成文本样本

8. 部分函数

   ```python
   optimizer = torch.optim.AdamW(model.parameters())  # 初始化optimizer
   optimizer.zero_grad()                               # 清空梯度
   torch.nn.functional.cross_entropy                   # 计算loss
   loss.backward()                                     # 计算梯度
   optimizer.step()                                    # 根据grad更新权重
   ```

9. decode策略用于生成更具原创性的文本

10. **贪心解码**：初始LLM在生成过程中的每一步，都会选取词汇表中概率得分最高的token作为生成的token（argmax）

11. **Temperature scaling**：对同一个50257的概率分布，大量重复采样，采样概率与概率分布一致（multinomial）
    - 将输出logits除以一个temp值后再softmax
      - `temp > 1`：logits都变小，差异变小，生成的文本多样性增加，但也更可能出现无意义的内容
      - `temp < 1`：logits都变大，差异变大，差异变得更加显著

12. **top-k**：将采样限制在最有可能的前k个token内，并通过将其他token的概率设为零，将它们排除在选择之外

## 六、用于分类任务的微调

1. 分类微调模型可以视为高度专业化的模型，而相比之下，开发一个适用于各种任务的通用型模型通常更具挑战性

2. 大概流程：数据集准备 → 模型设置 → 模型微调与评估

3. 修改输出层，将输出从50257维的向量修改为二维向量

4. 并不需要对所有模型层进行微调。这是因为，在基于神经网络的语言模型中，低层通常捕捉到的是基本的语言结构和语义，这些特征适用于多种任务和数据集。因此，只微调最后几层（接近输出层），它们更专注于细致的语言模式和任务特定的特征，通常就足够使模型适应新任务。

## 七、指令遵循微调

低秩适应（LoRA），这是最广泛使用的参数高效微调技术之一
